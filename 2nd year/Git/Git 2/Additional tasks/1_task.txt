Поработайте в группе по методологии GitHubFlow.

Повторите этапы классных задач №2-3, только заменив имя папки на git_lab3_lesson2, а имя репозитория на githubflow.
Распределите между собой ветки так, чтобы каждый из вас попробовал «зарелизить» (то есть выпустить) как минимум 1 ветку.
Перейдите в веб-интерфейс GitHub на страницу ветки, нажмите на кнопку «New pull request».
Убедитесь, что указаны верные ветки для создания пулл-реквеста:
base — ветка куда сливать (master),
compare — ветка, которую надо слить — ваша ветка, которую вы хотите «выпустить в продакшн».
Напишите заголовок пулл-реквеста и краткое описание, нажмите «Create pull request».
Пулл-реквест создан, теперь можете:
проверить коммиты, входящие в пулл-реквест (вкладка commits),
посмотреть изменения (вкладка Files Changed) или
принять пулл-реквест (вкладка Conversation, кнопка Merge pull request).
Примите пулл-реквест и повторите его для остальных веток.
Если у вас возникли конфликты, то их надо решить. Как это сделать, можно прочитать в официальной документации по GitHubFlow
